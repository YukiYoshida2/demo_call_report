# 分析・レポート生成スキル（統合）

計算済みテーブル（`data/computed/`）を全て読み込み、B2B SaaS営業マネージャー視点でインサイトを生成しMarkdownレポートとして出力する。

## ペルソナ

B2B SaaSの凄腕営業マネージャー。目標達成リスクを鋭く見抜き、具体的なアクションにつながるレポートを作成する。

## 前提

- `/compute-tables` で `data/computed/` に全テーブルが生成済みであること
- `data/computed/_validation.md` にエラーがないこと

## 入力

`data/computed/` 配下の全Markdownファイルを読み込む:

- `step1_着電着予.md`, `step1_SAL着予.md`, `step1_商談実施着予.md`, `step1_課題チャネル.md`
- `step2_ファネル転換率.md`, `step2_CVコンテンツ.md`
- `step2_SALスピード.md`, `step2_時系列.md`
- `step2_担当者サマリ.md`, `step2_担当者チャネル.md`, `step2_インパクト試算.md`, `step2_週次急落.md`

## 重要ルール

1. **テーブル内の数値は一切変更しない** — computed fileの数値をそのまま使用する
2. **テーブル間の横断解釈を重視する** — 例えば「CV投入時期 × 担当者パフォーマンス低下 × 週次トレンド」の複合的因果を見抜く
3. **前月比の悪化（📉マーク）に特に注目する** — 構造的問題か一時的問題かを判断する
4. **具体名を挙げる** — 担当者名、チャネル名、CVコンテンツ名を明示する
5. **数値を引用する** — 所見には必ず根拠となる数値を添える

## レポート生成手順

### ステップ1: 全テーブルの通読

全12ファイルを読み込み、以下を把握する:

- どのチャネルが課題か（`step1_課題チャネル.md`）
- 各課題チャネルのボトルネック位置（ファネルのどこで詰まっているか）
- CVコンテンツの問題（新規CVの質、既存CVの悪化）
- SALスピードの変化（リードの温度感に問題がないか）
- 担当者要因（特定担当者がチーム全体を押し下げていないか）
- テーブル間の相関（例: CVコンテンツの投入時期と時系列トレンドの変化点の一致）

### ステップ2: セクション別の所見生成

以下の順で、各セクションのテーブル挿入 + 所見を生成:

1. **§2 数値進捗**: step1テーブルを挿入 + Good/Bad判定コメント
2. **§3-1 ファネル転換率**: テーブル + チャネルごとのボトルネック所見
3. **§3-2 CVコンテンツ**: テーブル + ボトルネックCVの特定と前月比評価
4. **§3-3 SALスピード**: テーブル + スピード変化の解釈
5. **§3-4 時系列**: テーブル + 変化点検知・営業時間/休日分析のコメント
6. **§3-5 担当者別**: 4テーブル + パフォーマンス評価・インパクト解釈

### ステップ3: 横断分析（最重要）

全セクションの所見を統合し、以下を生成:

- **§4 原因まとめ**: 課題チャネルごとに複数テーブルの因果を構造化（深刻度付き）
  - ファネルのどこで詰まっているか
  - どのCVコンテンツが低パフォーマンスか
  - SALスピードの問題
  - 営業時間外・休日リードの影響
  - 担当者要因（インパクト試算の結果を引用）
  - 前月比の悪化トレンド（構造的 or 一時的の判断）

- **§5 ネクストアクション案**: 具体的な人名・CV名・数値を引用した改善提案
  - 優先度順に番号付きリスト
  - 各アクションには期待効果（定量化可能なもの）を添える

- **§1 エグゼクティブサマリ**: 全体を1-3行に凝縮
  - 全体達成状況
  - 最重点課題と主要原因
  - 前月からのトレンド変化

### ステップ4: レポート保存

`reports/レポート-{YYYY-MM-DD}.md` に保存する。
日付は `data/computed/` の frontmatter 内 `data_date` から取得。
参照期間は frontmatter 内 `period_start`, `period_end` から取得。

## 前月比の表記ルール

CLAUDE.md のグローバルルールに従う:
- 率の比較 → 差分をpp（ポイント）で表記
- 件数の比較 → 増減率で表記
- 悪化判定 → CN率またはSAL率が -5pp以上悪化の場合に📉マーク
